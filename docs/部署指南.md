# 部署指南：从拿到代码到上线运行

## 第一步：获取代码

### 方式 A：直接下载（推荐）

项目文件位于 `/mnt/agents/output/asset-allocation-agent/`，你只需要下载这些核心文件：

```
asset-allocation-agent/
├── backend/
│   ├── main.py                          ← 后端入口
│   ├── .env                             ← 配置文件（需修改）
│   └── app/                             ← 业务代码
│       ├── core/config.py               ← 配置管理
│       ├── core/llm_client.py           ← LLM客户端
│       ├── models/user_profile.py       ← 用户画像模型
│       ├── routers/chat.py              ← 对话API
│       ├── routers/plan.py              ← 方案API
│       ├── services/state_machine.py    ← KYC状态机
│       ├── services/allocation_engine.py ← 配置引擎
│       ├── services/stress_test.py      ← 压力测试
│       ├── services/data_service.py     ← 数据服务
│       └── services/session_manager.py  ← 会话管理
├── frontend/
│   └── index.html                       ← 前端页面（单文件）
└── docs/
    └── README.md                        ← 项目文档
```

在你的电脑上新建文件夹，把上面的文件按目录结构放好。

## 第二步：后端部署

### 2.1 安装 Python 环境

**前提**：电脑已安装 Python 3.10+

```bash
cd backend

# 创建虚拟环境（隔离依赖）
python -m venv venv

# 激活虚拟环境
# Mac/Linux:
source venv/bin/activate
# Windows:
# venv\Scripts\activate

# 安装依赖
pip install fastapi uvicorn dashscope akshare numpy pandas pydantic python-dotenv
```

> 看到 `(venv)` 前缀说明虚拟环境已激活

### 2.2 配置 API Key

编辑 `backend/.env` 文件：

```bash
# 打开 .env 文件，把 your-api-key-here 替换成你的真实 Key
DASHSCOPE_API_KEY=sk-你的真实APIKey
```

**如何获取 API Key**：
1. 访问 https://bailian.console.aliyun.com/
2. 用支付宝/阿里云账号登录
3. 左侧菜单 → "API Key 管理"
4. 点击 "创建新的 API Key"
5. 复制 Key 填入 `.env`

> 不填 Key 也能运行，系统会自动进入 Mock 模式（返回预设回复）

### 2.3 启动后端

```bash
# 确保在 backend 目录且虚拟环境已激活
python main.py
```

看到以下输出说明启动成功：

```
🚀 启动 AssetAllocationAgent...
📡 访问地址: http://0.0.0.0:8000
📚 API 文档: http://0.0.0.0:8000/docs
```

**验证**：浏览器打开 http://localhost:8000/docs 能看到 API 文档

## 第三步：前端部署（三种方式）

### 方式一：本地预览（最快，开发调试）

```bash
cd frontend

# 用 Python 临时 HTTP 服务器
```

**缺点**：关掉终端页面就停止

### 方式二：部署到 Vercel（推荐，免费+自动 HTTPS）

Vercel 是全球最大的前端托管平台，个人项目免费。

#### 步骤：

**1. 准备文件**

```bash
# 新建一个用于部署的文件夹
mkdir deploy-vercel && cd deploy-vercel

# 只需要复制 frontend/index.html
cp /path/to/asset-allocation-agent/frontend/index.html .

# 创建 Vercel 配置文件
cat > vercel.json << 'EOF'
{
  "version": 2,
  "routes": [
    { "handle": "filesystem" },
    { "src": "/.*", "dest": "/index.html" }
  ]
}
EOF
```

**2. 部署**

```bash
# 安装 Vercel CLI
npm i -g vercel

# 登录（会弹浏览器）
vercel login

# 部署
vercel --prod

# 按提示选择：
# ? Set up and deploy "..."? [Y/n]  → 回车
# ? Which scope do you want to deploy? → 选你的账号
# ? Link to existing project? [y/N]  → n（新建）
# ? What's your project name? → xiaocai-agent（自定义）
```

部署完成后会给你一个 HTTPS 链接：

```
🔍  Inspect: https://vercel.com/你的用户名/xiaocai-agent/xxxx
✅  Production: https://xiaocai-agent.vercel.app
```

**3. 修改 API 地址**

前端默认调用 `localhost:8000`，部署后需要改一下。

打开 `index.html`，找到这一行：

```javascript
const API_BASE = 'http://localhost:8000';
```

改成你的后端地址（后面会讲如何部署后端）：

```javascript
const API_BASE = 'https://你的后端地址';
```

然后重新部署：

```bash
vercel --prod
```

### 方式三：部署到腾讯云/阿里云对象存储（国内访问快）

#### 腾讯云 COS + CDN（推荐国内用户）

**1. 创建存储桶**
- 登录 https://console.cloud.tencent.com/cos
- 创建存储桶：
  - 名称：`xiaocai-frontend`
  - 地域：选离你用户最近的（如上海）
  - 访问权限：**公有读私有写**
- 记住你的**访问域名**（如 `xiaocai-frontend.cos.ap-shanghai.myqcloud.com`）

**2. 上传文件**

```bash
# 安装 COS CLI
pip install coscmd

# 配置（把 <> 里的换成你的真实信息）
coscmd config -a <SecretId> -s <SecretKey> \
  -b xiaocai-frontend -r ap-shanghai

# 上传
coscmd upload -r frontend/index.html /
```

或者直接在腾讯云控制台 → 存储桶 → 文件列表 → 上传文件

**3. 开启静态网站**
- 存储桶 → 基础配置 → 静态网站
- 开启静态网站
- 索引文档：`index.html`
- 错误文档：`index.html`
- 访问你的静态网站域名即可

**4. （可选）绑定自定义域名 + CDN**
- 存储桶 → 自定义 CDN 加速域名
- 添加你的域名（如 `agent.yourdomain.com`）
- 在域名服务商处添加 CNAME 记录
- 开启 HTTPS（免费证书）

## 第四步：后端部署（三种方式）

前端部署后，需要让后端也能被前端访问到。

### 方式一：本地开发（后端跑在本地电脑）

适合开发调试阶段：

```bash
# 后端
python main.py  # 跑在 localhost:8000

# 前端
cd frontend
python -m http.server 3000  # 跑在 localhost:3000
```

浏览器打开 http://localhost:3000 即可使用

**注意**：如果前端和后端端口不同，需要处理跨域（CORS），后端的 `main.py` 已经配置好了 `allow_origins=["*"]`

---

### 方式二：部署到阿里云 ECS / 腾讯云轻量服务器

**1. 买服务器**
- 阿里云：云服务器 ECS，选 `2核2G` 就够了，约 99元/年
- 腾讯云：轻量应用服务器，约 82元/年
- 系统选 **Ubuntu 22.04**

**2. 连接服务器**

```bash
ssh root@你的服务器IP
```

**3. 安装环境**

```bash
# 更新系统
apt update && apt upgrade -y

# 安装 Python
apt install python3 python3-pip python3-venv -y

# 创建项目目录
mkdir -p /opt/xiaocai && cd /opt/xiaocai

# 上传代码（在本地终端执行）
scp -r backend root@你的服务器IP:/opt/xiaocai/

# 回到服务器继续
ssh root@你的服务器IP
cd /opt/xiaocai/backend

# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 安装依赖
pip install fastapi uvicorn dashscope akshare numpy pandas pydantic python-dotenv

# 修改 .env，填入 API Key
vim .env
```

**4. 用 systemd 守护进程（后台运行）**

```bash
cat > /etc/systemd/system/xiaocai.service << 'EOF'
[Unit]
Description=Xiaocai Asset Allocation Agent
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/xiaocai/backend
Environment="PATH=/opt/xiaocai/backend/venv/bin"
ExecStart=/opt/xiaocai/backend/venv/bin/uvicorn main:app --host 0.0.0.0 --port 8000
Restart=always

[Install]
WantedBy=multi-user.target
EOF

# 启动
systemctl daemon-reload
systemctl enable xiaocai
systemctl start xiaocai

# 查看状态
systemctl status xiaocai

# 查看日志
journalctl -u xiaocai -f
```

**5. 配置 Nginx 反向代理（支持 HTTPS）**

```bash
apt install nginx -y

cat > /etc/nginx/sites-available/xiaocai << 'EOF'
server {
    listen 80;
    server_name api.yourdomain.com;  # 你的域名

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
EOF

ln -s /etc/nginx/sites-available/xiaocai /etc/nginx/sites-enabled/
rm /etc/nginx/sites-enabled/default
nginx -t
systemctl restart nginx
```

**6. 配置 HTTPS（免费 SSL 证书）**

```bash
apt install certbot python3-certbot-nginx -y
certbot --nginx -d api.yourdomain.com

# 按提示操作，完成后自动配置 HTTPS
```

现在你的后端地址是：`https://api.yourdomain.com`

## 常见问题

### Q1：前端页面空白怎么办？
- 检查浏览器控制台（F12 → Console）是否有报错
- 确认后端已启动且能访问 http://localhost:8000
- 检查 API_BASE 地址是否正确

### Q2：跨域报错（CORS）？
- 后端 `main.py` 中已配置 `allow_origins=["*"]`
- 生产环境应改为具体的域名：`allow_origins=["https://yourdomain.com"]`

### Q3：LLM 回复 "抱歉遇到了问题"？
- 检查 `.env` 中的 API Key 是否正确
- 检查阿里云账户是否有余额（新用户有免费额度）
- 不填 Key 也能用 Mock 模式测试流程

### Q4：如何更新代码？
- 本地修改 → 测试 → 重新部署
- Vercel：git push 自动部署
- ECS：scp 上传 + systemctl restart xiaocai
